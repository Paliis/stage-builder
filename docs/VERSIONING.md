# Версійність продукту (V0 / V1 / V2)

Це **продуктові** позначки для історії можливостей і зворотного зв’язку. Вони **не** замінюють semver у `package.json` (npm-версія лишається окремо).

| Позначка | Що означає |
|----------|------------|
| **V0** | Заморожений зріз: усе, що було **до** впровадження **налаштовуваного розміру площадки** (пресети розмірів поля, `fieldSizeM` у сцені та у файлі `*.stage.json`). |
| **V1** | Лінія **від зміни розміру площадки включно** до **межі з V2** — пресети поля, 2D-інструменти, розширення мішеней і реквізиту, документація нижче. *Замикання лінії V1* — напередодні уніфікації паперової палітри (див. V2). |
| **V2** | Поточна лінія: **від уніфікації паперових мішеней** (лише дві стійки, три висоти низу лиця для IPSC / A4 / Mini; прибрано одностійкові типи; міграція старих `*.stage.json`) **включно** — подальші зміни та відгуки. Опис V2 **доповнюємо** нижче по мірі релізів. |

**Зворотний зв’язок:** записи з **2026-04-06** до межі V2 відносимо до обліку **V1**; з моменту **фіксації V2** (документовано **2026-04-08**) — нові відгуки в контексті **V2**. Ідеї «на потім» поза поточним релізом — у [USER_FEEDBACK.md](./USER_FEEDBACK.md) (розділ накопичення для наступної хвилі).

## Перелік змін між версіями (огляд)

### V0 → V1

*Перехід від фіксованого/раннього поля до повноцінного редактора з налаштовуваною площадкою та розширеним набором об’єктів.*

| Тема | Зміст |
|------|--------|
| **Площадка** | Пресети розмірів поля, `FIELD_SIZE_LIMITS`, `fieldSizeM` у сторі та `*.stage.json`, підтвердження при зменшенні поля (`field.ts`, `stageStore`, `App.tsx`). |
| **2D** | Лінійки на краях вікна; вимір відстані (M, два кліки); шахматка 1 м на плані (узгоджено з [PLAN_FIELD_PDF_GRID.md](./PLAN_FIELD_PDF_GRID.md)); розстановка кліками після вибору типу; рамка виділення та копіювання/вставка (Ctrl+C/V). |
| **Мішені** | Mini IPSC (B3); метал на стійці 50 см / 1 м; паперові типи в т. ч. **одностійкові** IPSC / A4 / Mini (як у палітрі до V2). |
| **Реквізит** | Щити з портом (низький / високий / косий, дверцята); піраміда для зброї; дерев’яний стілець. |
| **Документація** | Розділи в [TECH.md](./TECH.md); перший варіант цього файлу версійності. |

### V1 → V2

*Уніфікація паперу під реальну конструкцію «дві стійки + три висоти лиця» та прибирання зайвих типів.*

| Тема | Зміст |
|------|--------|
| **Паперова палітра** | Лише **двостійкові** варіанти: `paperIpscTwoPost*`, `paperA4TwoPost*`, `paperMiniIpscTwoPost*` (Ground / Stand50 / Stand100). Типи **одностійкові** (`paperIpsc`, `paperA4`, `paperMiniIpsc`) **прибрані** з моделі та UI. |
| **Сумісність файлів** | При відкритті `*.stage.json`: `paperIpsc` / `paperA4` / `paperMiniIpsc` → відповідний `*TwoPostStand100`; збережено міграцію `paperIpscTwoPost` → `paperIpscTwoPostStand100` (`stageProjectFile.ts`). |
| **2D / 3D** | Підошви стійок і штрихи на плані; індикатори вздовж локального «низу» лиця; стійки в 3D під кутами силуету (B2 / A4 / Mini) (`targetSpecs.ts`, `StageCanvas.tsx`, `StageView3D.tsx`). |
| **Сцена** | **Undo / Redo** — кнопки біля вкладок 2D/3D, узгоджено з полями вводу (`zundo`, `App.tsx`). |
| **Інтерфейс** | Підписи без зайвої згадки «дві стійки» / «two posts»; оновлені рядки NS для паперу (`messages.ts`, тулбар). |
| **Документація** | Оновлення [FUNCTIONALITY.md](./FUNCTIONALITY.md), [TECH.md](./TECH.md), [README.md](../README.md); розділ про перевірку якості та бандл у TECH. |

Коротко: фіксовані або ранні габарити поля без вибору пресетів у UI; решта можливостей README на той момент (2D/3D, мішені, реквізит, PDF, `*.stage.json`, чернетка, PWA, UK/EN) — у межах того релізу.

Детальний список комітів тут не фіксуємо; за потреби межу можна відтворити в git за появою `FIELD_SIZE_PRESETS` / `fieldSizeM` у домені та сторі.

## V1 — накопичувальний опис (історія лінії до V2)

*Нижче — що входило в продукт протягом **V1** (до переходу на модель паперу V2). Частина пунктів про папер **оновлена** в V2: одностійкові типи замінено на двостійкові (див. таблицю V1→V2).*

- **Розмір площадки** — пресети розмірів поля, обмеження `FIELD_SIZE_LIMITS`, збереження в проєкті (`src/domain/field.ts`, `stageStore`, UI в `App.tsx`).
- **2D лінійки** — метричні осі закріплені на вікні перегляду (ліворуч і внизу), адаптивний крок поділок від 0,5 м; риски мають мінімальну довжину в px, щоб були видимі при масштабі «вся площадка» (`StageCanvas.tsx`, `drawViewportFixedRulers`).
- **Вимір відстані на 2D** — кнопка-лінійка на карті (над кошиком), клавіша **M**, два кліки, Esc, без 3D (`App.tsx`, `StageCanvas.tsx`, i18n).
- **Розстановка кліками** — вибір типу в тулбарі, потім кліки по плану для мішеней і реквізиту; Esc / повтор кнопки вимикають режим (`placementMode.ts`, `App.tsx`, `StageBuilderToolbar.tsx`, `StageCanvas.tsx`).
- **Документація** — розділ у [TECH.md](./TECH.md) про лінійки та вимір; цей файл версійності.
- **2D рамка та копіювання** — виділення зоною за центром об’єкта, внутрішній буфер + Ctrl+C/Ctrl+V, кнопки на карті (`StageCanvas.tsx`, `planClipboard.ts`, `App.tsx`).
- **Нові мішені (V1)** — паперова Mini IPSC (B3 номінал); металева пластина на стійці 50 см і 1 м (низ лиця в 3D; на плані як квадрат C3) — `models.ts`, `targetSpecs.ts`, `StageView3D.tsx`, i18n. *Папер IPSC / A4 / Mini також мали **одностійкові** пресети — у V2 замінено на лише двостійкові (див. вище).*
- **Щити з портом** — низький/високий порт 30×60 см; косий — паралелограм 7 см від кутів, планки AB/DC, сітка двома чотирикутниками (без `ExtrudeGeometry` holes); дверцята 30×30 — `propGeometry.ts`, `StageView3D.tsx`, `StageCanvas.tsx`, i18n.
- **Піраміда для зброї** — низ + верхній брусок **квадратного перерізу** (довжина вздовж X), трикутники, **труба** з виступом **20 см** над верхом — `WeaponRackPyramid3D`, `WEAPON_RACK_PIPE_ABOVE_TOP_M` (`StageView3D.tsx`); план — `drawWeaponRackPyramidPlan2D` (`StageCanvas.tsx`).
- **Стілець** — `woodChair`: `WOOD_CHAIR_*` у `propGeometry.ts`, `WoodChair3D`, `drawWoodChairPlan2D`, тулбар, `*.stage.json`.

## Для розробників

- Щоб не плутати з **форматом файлу** вправи: у коді є `STAGE_PROJECT_VERSION` у `stageProjectFile.ts` — це версія **схеми JSON**, не V0/V1/V2 UI.
- У реліз-нотах / комунікації з користувачами можна посилатися на **V2** як на поточну гілку після уніфікації паперових мішеней.
