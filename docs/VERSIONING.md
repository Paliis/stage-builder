# Версійність продукту (V0 / V1 / V2)

Це **продуктові** позначки для історії можливостей і зворотного зв’язку. Вони **не** замінюють semver у `package.json` (npm-версія лишається окремо). Загальний бізнес- і технічний опис продукту — у **[PRODUCT.md](./PRODUCT.md)**. Ідеї на майбутнє та грумінг — у **[BACKLOG.md](./BACKLOG.md)**.

| Позначка | Що означає |
|----------|------------|
| **V0** | Заморожений зріз: усе, що було **до** впровадження **налаштовуваного розміру площадки** (пресети розмірів поля, `fieldSizeM` у сцені та у файлі `*.stage.json`). |
| **V1** | Лінія **від зміни розміру площадки включно** до **межі з V2** — пресети поля, 2D-інструменти, розширення мішеней і реквізиту, документація нижче. *Замикання лінії V1* — напередодні уніфікації паперової палітри (див. V2). |
| **V2** | Поточна лінія: **від уніфікації паперових мішеней** (лише дві стійки, три висоти низу лиця для IPSC / A4 / Mini; прибрано одностійкові типи; міграція старих `*.stage.json`) **включно** — подальші зміни та відгуки. Опис V2 **доповнюємо** нижче по мірі релізів. У **2026-04** до лінії V2 додано **хмарні посилання на вправу** (share **`/v/`** / **`/e/`**, публікація, PDF QR, noindex, OG для ботів) — **[BL-001](./BL-001_SHARE_LINK_PLAN.md)**; **edit token** і розширення поза MVP — у [BACKLOG.md](./BACKLOG.md). |

**Зворотний зв’язок:** записи з **2026-04-06** до межі V2 відносимо до обліку **V1**; з моменту **фіксації V2** (документовано **2026-04-08**) — нові відгуки в контексті **V2**. Ідеї «на потім» поза поточним релізом — у [USER_FEEDBACK.md](./USER_FEEDBACK.md) (розділ накопичення для наступної хвилі).

## Повний аналіз версійності (V0 / V1 / V2)

Це **дві різні ролі** одних і тих самих позначок:

1. **Для історії коду й документації** — **V0 → V1 → V2** покроково ділять *коли* з’явилися можливості (див. таблиці нижче та накопичувальні списки).
2. **Для повідомлення користувачам зараз** — використовуємо **один накопичувальний текст під маркером V2** у розділі [«Повідомлення для користувачів: оновлення V2»](#повідомлення-для-користувачів-оновлення-v2): туди входить **усе**, що ви вже оголошували раніше окремими нотами (розмір поля, 2D-інструменти, мішені, реквізит, оновлення застосунку), **плюс** зміни по полю / екрану / фону / PDF / сітці **плюс** останні зміни по **стійках і паперових мішенях**. Так користувач не «отримує знову V1» — він отримує **повний зріз продукту**, який ми маркуємо як хвилю **V2**.

| Позначка | Що це означає в аналізі | Що потрапляє в «комунікацію V2» |
|----------|-------------------------|----------------------------------|
| **V0** | До пресетів і `fieldSizeM`: раннє поле без того UI. | Лише як контекст; **не** виносимо окремим реліз-листом для користувачів. |
| **V1** (лінія в продукті) | Від налаштовуваного поля до межі перед уніфікацією паперу: 2D/3D, сітка, PDF-режим, фон, великі екрани, нові мішені й реквізит тощо. | **Усі** ці пункти включені в **єдине повідомлення V2** (див. нижче), а не окремий список «лише V1». |
| **V2** (лінія в продукті) | Від уніфікації паперу (дві стійки, три висоти, міграція файлів) і подальших змін. | Те саме повідомлення V2 описує і «старі» можливості редактора, і **нове** по стійках / плану / Undo / підписах. |

**Коротко:** *внутрішньо* V1 і V2 розведені по таблицях; *зовнішньо* для розсилки — **один список «оновлення V2»**, без дублювання окремого «що нового у V1» як головного каналу.

## V1 — накопичувальний опис (історія лінії до V2)

*Нижче — що входило в продукт протягом **V1** (до переходу на модель паперу V2). Частина пунктів про папер **оновлена** в V2: одностійкові типи замінено на двостійкові (див. таблицю V1→V2).*

- **Розмір площадки** — пресети розмірів поля, обмеження `FIELD_SIZE_LIMITS`, збереження в проєкті (`src/domain/field.ts`, `stageStore`, UI в `App.tsx`).
- **2D лінійки** — метричні осі закріплені на вікні перегляду (ліворуч і внизу), адаптивний крок поділок від 0,5 м; риски мають мінімальну довжину в px, щоб були видимі при масштабі «вся площадка» (`StageCanvas.tsx`, `drawViewportFixedRulers`).
- **Вимір відстані на 2D** — кнопка-лінійка на карті (над кошиком), клавіша **M**, два кліки, Esc, без 3D (`App.tsx`, `StageCanvas.tsx`, i18n).
- **Розстановка кліками** — вибір типу в тулбарі, потім кліки по плану для мішеней і реквізиту; Esc / повтор кнопки вимикають режим (`placementMode.ts`, `App.tsx`, `StageBuilderToolbar.tsx`, `StageCanvas.tsx`).
- **Документація** — розділ у [TECH.md](./TECH.md) про лінійки та вимір; цей файл версійності.
- **2D рамка та копіювання** — виділення зоною за центром об’єкта, внутрішній буфер + Ctrl+C/Ctrl+V, кнопки на карті (`StageCanvas.tsx`, `planClipboard.ts`, `App.tsx`).
- **Нові мішені (V1)** — паперова Mini IPSC (B3 номінал); металева пластина на стійці 50 см і 1 м (низ лиця в 3D; на плані як квадрат C3) — `models.ts`, `targetSpecs.ts`, `StageView3D.tsx`, i18n. *Папер IPSC / A4 / Mini також мали **одностійкові** пресети — у V2 замінено на лише двостійкові (див. вище).*
- **Щити з портом** — низький/високий порт 30×60 см; косий — паралелограм 7 см від кутів, планки AB/DC, сітка двома чотирикутниками (без `ExtrudeGeometry` holes); дверцята 30×30 — `propGeometry.ts`, `StageView3D.tsx`, `StageCanvas.tsx`, i18n.
- **Піраміда для зброї** — низ + верхній брусок **квадратного перерізу** (довжина вздовж X), трикутники, **труба** з виступом **20 см** над верхом — `WeaponRackPyramid3D`, `WEAPON_RACK_PIPE_ABOVE_TOP_M` (`StageView3D.tsx`); план — `drawWeaponRackPyramidPlan2D` (`StageCanvas.tsx`).
- **Стілець** — `woodChair`: `WOOD_CHAIR_*` у `propGeometry.ts`, `WoodChair3D`, `drawWoodChairPlan2D`, тулбар, `*.stage.json`.
- **Покриття площадки в 3D** — вибір **земля / трава / пісок** (`fieldGroundCover3d`), зберігається в сцені та `*.stage.json` (`fieldGround3d.ts`, `StageView3D`, `stageStore`).
- **3D «як у PDF»** — режим камери з тим самим співвідношенням сторін, що й PNG для брифінгу; рамка кадру в UI (`CameraMode3D` `pdf`, `a4PrintLayout.ts`).
- **Адаптивний layout** — зручніша картка плану на великих моніторах (`App.css`, див. [PLAN_FIELD_PDF_GRID.md](./PLAN_FIELD_PDF_GRID.md)).

## Повідомлення для користувачів: оновлення V2

**Один текст для розсилки чи екрана «Що нового»** — формулювання **без технічних деталей**. Сюди **свідомо зведено все**: і те, що раніше виходило окремими повідомленнями про можливості редактора (у т. ч. ваш приклад з пресетами **15×20** / **20×20**, лінійками, мішенями…), і зміни по **полю, великим екранам, фону, кадру PDF, сітці**, і **останні зміни по стійках і паперових мішенях** (продуктова лінія V2 у таблицях вище). Користувачу **не потрібно** збирати картину з «V1» і «V2» — це **єдиний актуальний перелік**.

### Повний перелік змін (копіювати цілком)

**Площадка та вікно**

- **Розмір поля** — зручні **пресети**, зокрема **15×20** та **20×20** метрів; можна задати **власні ширину й довжину** в метрах — до **50 м** по одній осі й до **100 м** по іншій (дуже великі майданчики).
- **Великі екрани** — на широкому вікні план і тривимірний вигляд **краще заповнюють екран**, без зайвих «порожніх смуг» по краях.

**План 2D**

- **Лінійки** — зліва й знизу показуються **метри**, легше орієнтуватися без підрахунку клітинок.
- **Сітка** — основна сітка з кроком **0,5 м** і **шахматка 1 м** на великих полях.
- **Вимір відстані** — режим лінійки (кнопка або **M**): два кліки по плану — видно **відстань у метрах**.
- **Розстановка кліками** — обираєте тип мішені чи реквізиту на панелі й ставите об’єкти кліками; **Esc** вимикає режим.
- **Виділення та копіювання** — виділити зону на плані й скопіювати об’єкти (**Ctrl+C** / **Ctrl+V** або кнопки).

**3D і знімок «як для друку»**

- **Фон площадки** — вибір **земля / трава / пісок**; зберігається у вправі.
- **Кадр як для PDF** — у 3D є режим з **тим самим співвідношенням сторін**, що й **знімок для брифінгу / друкованого PDF**; видно **рамку кадру**, що потрапить на «картинку».

**Мішені та реквізит (розширення палітри)**

- **Mini IPSC** (паперова); **металеві пластини** на стійці **50 см** і **1 м**.
- **Реквізит** — щити з портом (низький, високий, косий); **піраміда для зброї**; **дерев’яний стілець** із зоною під зброю.

**Паперові мішені, стійки та останні оновлення (ядро змін V2)**

- **Одна схема для паперу** — як у реальній конструкції: **дві стійки** і **три варіанти висоти** низу лиця (на землі, на низькій стійці ~50 см, на високій ~100 см) для **IPSC**, **A4** та **Mini**; зайві дублікати в палітрі прибрано.
- **Старі файли** — збережені раніше вправи **відкриваються без ручної переробки**: типи паперових мішеней **підставляються автоматично** під нову схему.
- **План і 3D** — для паперових мішеней наочніше показані **стійки**, **підошви** та **орієнтація** лиця на 2D і в 3D.
- **Редагування** — **Скасувати** та **Повторити** біля перемикача **2D / 3D**.
- **Підписи** — назви та підказки для паперу **простіші й коротші**.

**Застосунок**

- **Оновлення** — якщо вийшла нова версія, з’явиться **підказка**: оновитися в зручний момент, а не «тихе» перезавантаження сторінки.

### Примітка для команди

Попередні короткі оголошення (перша хвиля) лишаються валідними за змістом — їх **об’єднано** в цьому тексті. Надалі для користувачів оновлюйте **цей** блок; розділення на окремі списки «лише V1» / «лише V2» для розсилки **не** використовуємо.
