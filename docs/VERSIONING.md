# Версійність продукту (V0 / V1)

Це **продуктові** позначки для історії можливостей і зворотного зв’язку. Вони **не** замінюють semver у `package.json` (npm-версія лишається окремо).

| Позначка | Що означає |
|----------|------------|
| **V0** | Заморожений зріз: усе, що було **до** впровадження **налаштовуваного розміру площадки** (пресети розмірів поля, `fieldSizeM` у сцені та у файлі `*.stage.json`). |
| **V1** | Поточна лінія: **від зміни розміру площадки включно** і далі — нові можливості та правки за зворотним зв’язком користувачів. Опис V1 **доповнюємо** в розділі нижче по мірі релізів. |

**Зворотний зв’язок:** записи в [USER_FEEDBACK.md](./USER_FEEDBACK.md) з дати початку систематичного збору (**2026-04-06**) відносять до планування/обліку **V1** (попередній досвід користувачів формально входить у «довоєнний» V0, якщо не потрапив у код після межі V1).

---

## V0 (базовий зріз, до пресетів розміру поля)

Коротко: фіксовані або ранні габарити поля без вибору пресетів у UI; решта можливостей README на той момент (2D/3D, мішені, реквізит, PDF, `*.stage.json`, чернетка, PWA, UK/EN) — у межах того релізу.

Детальний список комітів тут не фіксуємо; за потреби межу можна відтворити в git за появою `FIELD_SIZE_PRESETS` / `fieldSizeM` у домені та сторі.

---

## V1 — накопичувальний опис

*Оновлюйте цей список при значущих змінах для користувачів (один або кілька пунктів на логічний блок).*

- **Розмір площадки** — пресети розмірів поля, обмеження `FIELD_SIZE_LIMITS`, збереження в проєкті (`src/domain/field.ts`, `stageStore`, UI в `App.tsx`).
- **2D лінійки** — метричні осі вздовж краю поля та вздовж `y = 0`, адаптивний крок поділок від 0,5 м; риски мають мінімальну довжину в px, щоб були видимі при масштабі «вся площадка» (`StageCanvas.tsx`, `drawFieldRulers`).
- **Вимір відстані на 2D** — кнопка-лінійка на карті (над кошиком), клавіша **M**, два кліки, Esc, без 3D (`App.tsx`, `StageCanvas.tsx`, i18n).
- **Розстановка кліками** — вибір типу в тулбарі, потім кліки по плану для мішеней і реквізиту; Esc / повтор кнопки вимикають режим (`placementMode.ts`, `App.tsx`, `StageBuilderToolbar.tsx`, `StageCanvas.tsx`).
- **Документація** — розділ у [TECH.md](./TECH.md) про лінійки та вимір; цей файл версійності.
- **2D рамка та копіювання** — виділення зоною за центром об’єкта, внутрішній буфер + Ctrl+C/Ctrl+V, кнопки на карті (`StageCanvas.tsx`, `planClipboard.ts`, `App.tsx`).
- **Нові мішені** — паперова Mini IPSC (B3 номінал); металева пластина на стійці 50 см і 1 м (низ лиця в 3D; на плані як квадрат C3) — `models.ts`, `targetSpecs.ts`, `StageView3D.tsx`, i18n.
- **Щити з портом** — низький / високий / косий порт; щит із дверцями в порті (дерев’яна панель + ручка) — `PropType`, `propGeometry.ts` (`isShieldWithPortFamily`, косий отвір на плані), `StageView3D.tsx`, `StageCanvas.tsx`, i18n.

## Для розробників

- Щоб не плутати з **форматом файлу** вправи: у коді є `STAGE_PROJECT_VERSION` у `stageProjectFile.ts` — це версія **схеми JSON**, не V0/V1 UI.
- У реліз-нотах / комунікації з користувачами можна посилатися на **V1** як на «поточну гілку можливостей після вибору розміру поля».
