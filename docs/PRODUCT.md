## Частина A. Бізнес-опис

### A.1. Назва та позиціонування

**Stage Builder** — безкоштовний веб-застосунок для **проєктування стрільбищних вправ у стилі IPSC**: розстановка мішеней і реквізиту на **метричному плані**, перевірка сцени в **3D**, заповнення **брифінгу** та експорт **PDF** зі знімком сцени та QR-кодом. Працює в браузері, підтримує **українську та англійську**, може працювати **офлайн** після встановлення як **PWA**.

Публічний екземпляр порталу: **[shooters-tools.com](https://shooters-tools.com)** (редактор Stage Builder: **[…/stage-builder](https://shooters-tools.com/stage-builder)**). Попередній хост проєкту на Vercel редіректить на канонічний домен.

### A.2. Цільова аудиторія

- **Організатори та РО** — швидко накреслити схему майданчика й передати її учасникам.
- **Тренери та клуби** — готувати зрозумілі плани та брифінги без спеціалізованого настільного софта.
- **Стрілки** — переглянути сцену в 3D, оцінити відстані та розташування мішеней.

Продукт **не** замінює офіційні регламенти IPSC, рішення арбітражу чи сертифіковане ПЗ для змагань — він **допомагає** готувати матеріали та зменшувати непорозуміння між людьми.

### A.3. Проблема та рішення

| Потреба | Як Stage Builder допомагає |
|--------|----------------------------|
| Узгодити **розміри майданчика** й об’єкти в метрах | План з сіткою, лінійками, виміром відстані, пресетами та довільним розміром поля в межах продукту |
| Показати **як це виглядає** для стрілка | 3D-перегляд, фон площадки, режим кадру **«як у PDF»** |
| **Передати** сцену іншим | Файл `*.stage.json`, PDF брифінгу, чернетка в браузері |
| Працювати **без встановлення** або **офлайн** | Відкрити URL; PWA з кешем і підказкою оновлення |

### A.4. Ключова цінність (value proposition)

- **Швидкість** — вибір типу в панелі, **кліки по плану** для розстановки; undo/redo для сцени.
- **Точність орієнтиру** — метричні лінійки, сітка 0,5 м і шахматка 1 м, вимір **M** між двома точками.
- **Сумісність із реальністю** — паперові мішені з **двома стійками** і **трьома висотами** лиця; міграція старих файлів при відкритті.
- **Документ для залу** — PDF з полями брифінгу, знімком 3D і QR на застосунок.
- **Доступність** — безкоштовно в браузері; двомовний інтерфейс.

### A.5. Основні сценарії (користувацькі потоки)

1. **Створити вправу** — задати розмір поля → розставити мішені й реквізит → за потреби перевірити 3D і кадр для PDF → заповнити брифінг → експортувати PDF або зберегти `*.stage.json`.
2. **Відкрити готову вправу** — імпорт JSON; застарілі типи мішеней підставляються автоматично (див. FUNCTIONALITY / TECH).
3. **Продовжити пізніше** — чернетка сцени й брифінгу в `localStorage` між візитами (окремо від файлу на диску).
4. **Оновити застосунок** — PWA: банер «є нова версія», оновлення за згодою користувача (без примусового перезавантаження).

### A.6. Монетизація та підтримка

Проєкт позиціонується як **безкоштовний** для користувачів кінцевого застосунку. Канали зворотного зв’язку та опційної підтримки — у [README.md](../README.md) (email, Telegram, Monobank). Збір ідей для розвитку — [USER_FEEDBACK.md](./USER_FEEDBACK.md).

### A.7. Обмеження та застереження (бізнес-рівень)

- Оцінка **мінімальних пострілів** у UI — **орієнтовна**, не заміна правил змагання.
- **Кути безпеки** на плані — візуальна підказка, не юридично значуща перевірка.
- Синхронізація між пристроями **лише** через експорт/імпорт файлу або власні процеси користувача — хмарного акаунта в продукті немає.

## Частина B. Технічний опис

### B.1. Стек технологій

| Шар | Технології |
|-----|------------|
| UI | React 19, TypeScript |
| Стан | Zustand; undo/redo — `zundo` (temporal) для **сцени** |
| Збірка | Vite 8, `vite-plugin-pwa` |
| 2D | Canvas 2D (`StageCanvas`) |
| 3D | Three.js, React Three Fiber, Drei (lazy-завантаження 3D) |
| PDF | jsPDF, jspdf-autotable, qrcode; шрифти — вбудований Roboto для кирилиці |
| Тести | Vitest |
| Якість | ESLint; `npm run check` = lint + тести + production build (як у CI) |
| Деплой | GitHub Actions (CI), Vercel (production з `main`, preview з PR) |

### B.2. Архітектура (шари)

| Шар | Каталог | Відповідальність |
|-----|---------|------------------|
| **Domain** | `src/domain/` | Типи мішеней і реквізиту, геометрія, габарити IPSC/A4/Mini, підрахунки, парсинг/збірка `*.stage.json`, буфер плану, кути безпеки, координати 3D, покриття площадки, макет A4/PDF |
| **Application** | `src/application/` | Zustand: сцена (`stageStore`), брифінг (`briefingStore`), чернетка сесії (`sessionDraft`), персист чернетки |
| **Presentation** | `src/presentation/` | План, 3D, тулбар, міні-карта, PDF-експорт, PWA-банер, аналітика |
| **i18n** | `src/i18n/` | Повідомлення UK/EN |

Точка входу: `src/main.tsx` → гідратація чернетки, PWA, `I18nProvider`, `App`.

### B.3. Дані та формати

- **Файл вправи** — `*.stage.json`: версія схеми (`STAGE_PROJECT_VERSION` у коді), знімок сцени (`fieldSizeM`, `fieldGroundCover3d`, `weaponClass`, `targets`, `props`) + об’єкт брифінгу. Деталі й міграції — [TECH.md](./TECH.md).
- **Чернетка** — `localStorage`, debounce; при очищенні вправи скидається разом із сценою.
- **Брифінг** — окремий стор **без** undo; undo стосується лише сцени (мішені, реквізит, поле, покриття 3D, ім’я, клас зброї).

### B.4. Ключові підсистеми (коротко)

| Підсистема | Призначення |
|------------|-------------|
| **2D-план** | Сітка, snap, лінійки, вимір, рамка, копіювання/вставка, розстановка з тулбару |
| **3D** | Камери огляд / зона стрільця / **pdf** (той самий aspect, що й знімок у PDF); тіні; покриття землі |
| **PDF** | Таблиця брифінгу A4, PNG з 3D, QR, типографіка |
| **PWA** | Service worker `prompt`, банер оновлення з обмеженням частоти |
| **Безпека сцени** | Парсинг кутів з брифінгу, клин на плані, попередження по мішенях поза сектором |

### B.5. Якість, CI та середовища

- **CI** (GitHub Actions): на push/PR у `main` і `staging` — `npm run check`.
- **Staging**: окремий Vercel-проєкт, гілка `staging`, опційно `VITE_SITE_ENV=staging` (noindex, стрічка в UI) — деталі в [TECH.md](./TECH.md).
- **Аналітика**: Vercel Analytics у коді; GA4 — лише за `VITE_GA_MEASUREMENT_ID`.

### B.6. Продуктові версії (V0 / V1 / V2) і комунікація

Позначки **V0 / V1 / V2** описують **еволюцію можливостей** (історія фіч, зворотний зв’язок), **не** semver npm. Для **одного накопичувального повідомлення користувачам** використовується текст **«оновлення V2»** у [VERSIONING.md](./VERSIONING.md) — він об’єднує можливості редактора та зміни по паперових мішенях і стійках.

*English (short): Stage Builder is a free browser-based IPSC-style stage designer with a metric 2D plan, 3D preview, bilingual UI, JSON project files, PDF briefing export, and PWA offline support. Business-wise it targets match staff, clubs, and shooters; technically it uses React 19, TypeScript, Vite, Canvas 2D, Three.js/R3F, Zustand+zundo, and is deployed on Vercel with GitHub Actions CI.*
