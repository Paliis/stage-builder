# Промпт для агента: аудит ФПСУ vs IPSC і план заповнення карток RO Helper

Скопіюй блок нижче в новий чат (або передай як системний/користувацький контекст). Репозиторій: **Stage Builder**, модуль контенту: `content/ro-helper/`.

---

## Контекст і мета

Ти працюєш у репозиторії **Stage Builder**. У модулі **RO Helper** кожна картка — це пара markdown-файлів **`uk`** + **`en`** за шляхом:

`content/ro-helper/{locale}/{discipline}/{category}/{slug}.md`

Повний перелік карток — **`docs/RO_HELPER_CARD_MATRIX.csv`** (308 рядків, **C26–C333**). Індекс для людей: **`content/ro-helper/INDEX.md`**.

**Базис правил:** офіційні PDF **IPSC Jan 2026** (у кожного файлу картки є `primary_url` на відповідний rulebook).

**Національний шар (Україна):** правила **ФПСУ** — орієнтир для мапінгу: **`docs/RO_HELPER_IPSC_FPSU_TOPIC_MAP.md`** (посилання на [Правила ФПСУ 2020 на upsf.org.ua](https://upsf.org.ua/rules/upsf/2020/index?type=rules)).

**Продуктова спека:** **`docs/RO_HELPER_V0.md`** — особливо **§4** і **§4.1**: блок «Локально (ФПСУ)» у картці **не вигадується**; `fpsu_delta_verified: true` ставиться **лише** після перевірки офіційним документом ФПСУ і узгодження формулювань (рев’юер). Якщо відмінність не підтверджена — залишається лише шар IPSC, `fpsu_delta_verified` лишається `false`.

## Завдання 1 — Зафіксувати розходження (дельти)

1. **Побудуй матрицю «IPSC ↔ ФПСУ»** не для всього тексту закону, а для **тем, які покриті картками** (за `category` + якорями з CSV колонки `ipsc_anchor_hint` / змістом карток):
   - Де **збігається** суть, але **різна нумерація** або структура — зафіксуй окремо.
   - Де **є додаткові обмеження** ФПСУ (суворіші мінімуми, інші процедури) — з **цитатою або точним посиланням** на розділ Правил ФПСУ 2020.
   - Де **немає прямого аналога** у ФПСУ (тільки IPSC для міжнародного матчу) — познач «немає 1:1; локальний матч за регламентом організатора».

2. **Джерела:**
   - IPSC: PDF Jan 2026 по кожній з п’яти дисциплін (`handgun`, `pcc`, `rifle`, `mini_rifle`, `shotgun`) — URL з `primary_url` у відповідних картках або з topic map.
   - ФПСУ: актуальна веб-редакція Правил 2020; за потреби зазнач обмеження (якщо з’являться оновлення після 2020 — не підміняти без явного посилання на новий документ).

3. **Вихід артефакту 1:** таблиця (Markdown або CSV) з колонками на кшталт:
   - `card_id` / `slug` / `discipline` (якщо дельта саме для однієї дисципліни)
   - `ipsc_anchor` (п. правил IPSC)
   - `fpsu_section` (п. Правил ФПСУ 2020)
   - `delta_type`: `identical` | `stricter_local` | `wording_only` | `no_direct_mapping` | `needs_legal_review`
   - `summary_uk` (1–3 речення)
   - `evidence` (посилання на конкретний підрозділ ФПСУ)
   - `priority` (P1–P3 для заповнення карток)

## Завдання 2 — План заповнення інформації в картках

На базі таблиці дельт підготуй **план оновлення контенту** (без масової правки в одному PR, якщо обсяг великий — батчами за категоріями або за `discipline`).

Для **кожної** картки, де є підтверджена дельта, план має вказувати:

1. **Frontmatter**
   - `fpsu_refs`: масив об’єктів `{ rule, note, url? }` — тільки якщо є стабільне посилання або однозначна назва розділу.
   - `fpsu_delta_verified`: логіка переходу на `true` (хто підписав / яка умова).

2. **Тіло статті (обидві мови UK + EN)**
   - Розділ **«Локально (ФПСУ)»** / **«Local (FPSU)»**: коротко, без дублювання повного статуту; узгоджено з **§4** RO_HELPER_V0.
   - Якщо для EN-аудиторії формулювання інше за юридичним навантаженням — збережи **однаковий зміст**, різниця лише мовна.

3. **Порядок робіт**
   - Пріоритет: спочатку **safety** + **penalties** (DQ/зони), потім **scoring**, **equipment**, **match-admin** — або інше обґрунтування в плані.
   - Залежності: спочатку оновити **`docs/RO_HELPER_IPSC_FPSU_GLOSSARY.md`** / глосарій, якщо з’являються нові терміни ФПСУ.

4. **Ризики**
   - Де потрібна **юридична або федераційна** перевірка перед публікацією.
   - Де достатньо залишити дисклеймер «перевіряйте регламент матчу».

## Обмеження

- Не змінюй **`card_id`** і **`slug`** без окремого рішення (вони стабільні).
- Не помічай `fpsu_delta_verified: true`, поки немає явного підтвердження за **§4.1** RO_HELPER_V0.
- Парність **UK + EN** обов’язкова для кожної зміненої картки.
- Після суттєвих змін контенту — оновити **`content/ro-helper/INDEX.md`** (якщо змінюються нотатки до рядка картки) та узгодити з **`docs/RO_HELPER_CARD_MATRIX.csv`** (колонка `notes` / статуси за домовленістю команди).
