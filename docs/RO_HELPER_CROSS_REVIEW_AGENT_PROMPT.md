# Промпт для другого агента: перехресна перевірка RO Helper (покриття, якість, IPSC / ФПСУ)

Скопіюй блок нижче в **новий чат** (або передай як системний/користувацький контекст). Репозиторій: **Stage Builder**, модуль: **`content/ro-helper/`**. Ти — **другий рев’юер** (перехресна перевірка), не автор первинних чернеток.

Пов’язані артефакти (за потреби в тій же сесії): [RO_HELPER_FPSU_IPSC_REVIEW_AGENT_PROMPT.md](./RO_HELPER_FPSU_IPSC_REVIEW_AGENT_PROMPT.md) (аудит дельт і план карток), [RO_HELPER_FPSU_IPSC_DELTA_AND_PLAN.md](./RO_HELPER_FPSU_IPSC_DELTA_AND_PLAN.md).

## Контекст і мета

Ти працюєш у репозиторії **Stage Builder**. У модулі **RO Helper** кожна картка — пара **`uk` + `en`** markdown:

`content/ro-helper/{locale}/{discipline}/{category}/{slug}.md`

**Категорії підказок (SOS) для RO:** `safety` | `penalties` | `scoring` | `equipment` | `match-admin` — саме в цих рамках користувач шукає відповіді на лінії та в брифінгу.

**Повний перелік карток:** `docs/RO_HELPER_CARD_MATRIX.csv` (308 рядків карток × 5 дисциплін де застосовно; пари UK+EN = **616** файлів). Індекс: `content/ro-helper/INDEX.md`.

**Базис міжнародних правил:** **IPSC** (офіційні PDF **Jan 2026**); у тексті українських документів та розмовній практиці те саме часто позначають через **МКПС** (Міжнародна конфедерація практичної стрільби) — у картках канонічний якір нумерації та цитат: **IPSC**, `ipsc_refs`, `primary_url` на PDF обраної дисципліни.

**Національний шар (Україна):** **ФПСУ / UPSF** — [Правила ФПСУ 2020](https://upsf.org.ua/rules/upsf/2020/index?type=rules); мапінг глав: `docs/RO_HELPER_IPSC_FPSU_TOPIC_MAP.md`; словник EN↔UK: `docs/RO_HELPER_IPSC_FPSU_GLOSSARY.md`; таблиця дельт (орієнтир): `docs/RO_HELPER_FPSU_IPSC_DELTA_MATRIX.csv`.

**Спека продукту та контенту:** `docs/RO_HELPER_V0.md` (§2 рев’ю, §4–§4.1 шари IPSC/ФПСУ), `docs/RO_HELPER_CONTENT_TZ.md` (ТЗ статей, UK-термінологія).

**Інструменти в репо (для тебе або людини):**

- `npm run ro-helper:validate` — структура пакета vs матриця, наявність `ipsc_refs` / `fpsu_refs` з URL upsf, базові прапорці секцій.
- `npm run ro-helper:export-relevance` — ZIP з `relevance.csv` / `relevance.json` і формою **`REVIEW_FORM_UA.md`** для машинної + ручної фіксації висновків по кожній картці (`manual_*` колонки).

## Завдання 1 — Покриття тем (чи «все є», що може знадобитися RO)

Мета: перевірити, чи **набір карток** покриває типові **ситуації та питання**, які виникають у контексті **наших п’яти категорій** підказок, з урахуванням **п’яти дисциплін** IPSC (`handgun`, `pcc`, `rifle`, `mini_rifle`, `shotgun`).

1. Для кожної **`category`** побудуй короткий **чекліст сценаріїв RO** (наприклад: safety — AD, 180°, палець у скобі, DQ загальні, зони; penalties — процедура, FTE, walkthrough; scoring — методи, папір, оскарження; equipment — хроно, дивізіони, збої; match-admin — WSB, перестріли, протести). Познач: **є картка(и)** / **прогалина** / **перекриття** (дві картки про те саме без відмінності дисципліни/слогу).
2. Врахуй **міждисциплінні відмінності** IPSC (наприклад наявність **10.2.12** не у всіх PDF — див. `RO_HELPER_IPSC_FPSU_TOPIC_MAP.md` §2.2): чи немає карток, які підводять під правило, якого немає в rulebook дисципліни.
3. Вихід: таблиця або список **прогалин і перекритів** з пріоритетом **P1–P3** і посиланням на **`slug`** / `card_id` з матриці.

## Завдання 2 — Якість наповнення та релевантність

Для **вибірки** карток (мінімум **20** пар UK+EN, бажано з кожної `category` і з різних `discipline`) або для повного проходу за батчами:

1. **Релевантність:** чи заголовок, **Що це / What this is** і основний текст відповідають **`label_*` / темі `slug`** і не змішують дисципліни (наприклад текст PCC без зміни `primary_url`).
2. **Якість:** чи немає «води» замість дії RO; чи є чіткі кроки для RO; чи цифри/зони скорингу **звірені з PDF** цієї дисципліни (якщо сумнів — позначити «потрібна звірка з PDF», не вигадувати норму).
3. **UK+EN пара:** однаковий зміст і ті самі пункти IPSC; відмінності лише мовою ([RO_HELPER_V0 §2.1](./RO_HELPER_V0.md#21-чекліст-ревю-перед-релізом-статті) п.7). Перевірка UK на **«дуло»** — [RO_HELPER_CONTENT_TZ §1.1](./RO_HELPER_CONTENT_TZ.md#11-українська-термінологія-узгоджено-з-офіційним-стилем-фпсу).
4. Вихід: **звіт якості** (коротко по кожній перевіреній парі: OK / правки / критично) + **зведена статистика** (скільки OK, скільки потребують правок).

---

## Завдання 3 — Відповідність IPSC (МКПС) та ФПСУ

1. **IPSC:** чи `ipsc_refs` і цитати в секції **IPSC** відповідають **`primary_url`** для цієї **`discipline`** (Jan 2026); чи немає змішування rulebook.
2. **ФПСУ:** чи посилання в `fpsu_refs` і в блоці **«Локально (ФПСУ)»** / **«Local (FPSU)»** ведуть на **узгоджені розділи** Правил 2020; чи немає **вигаданих** відмінностей без первинника ([§4.1](./RO_HELPER_V0.md#41-хто-фіксує-відмінності-фпсу-від-ipsc)).
3. **МКПС у тексті:** якщо згадується національний контекст України, коректно пояснювати зв’язок **IPSC (міжнародні правила матчу)** і **ФПСУ (національні правила)**; не підміняти одне одним без формулювання змісту з документів.
4. Вихід: таблиця **«узгоджено / сумнів / суперечність»** з колонками: `card_id`, `slug`, `discipline`, тип проблеми (`ipsc_anchor` | `fpsu_link` | `uk_en_drift` | `terminology`), короткий опис, рекомендована дія.

## Обмеження

- Не змінюй **`card_id`** і **`slug`** без окремого рішення продукту.
- Не виставляй **`fpsu_delta_verified: true`**, якщо ти не уповноважений рев’юер згідно §4.1; у звіті лише **рекомендації**.
- Не публікуй довгі витяги rulebook без потреби; для звіту достатньо посилань і коротких цитат.
