# ТЗ: підготовка статей RO Helper (v0)

**Призначення:** операційне завдання для виробництва Markdown-статей модуля **RO Helper** за специфікацією [RO_HELPER_V0.md](./RO_HELPER_V0.md). Пайплайн: **модель C** — чернетка з LLM, **обов’язкове рев’ю** людиною перед публікацією.

**Останнє оновлення:** квітень 2026.

---

## 1. Результат роботи (deliverables)

1. Набір файлів **`*.md`** (одна стаття = один файл) з валідним **YAML frontmatter** згідно з [RO_HELPER_V0.md §2.2](./RO_HELPER_V0.md#22-поля-контенту-рекомендований-frontmatter).
2. Для кожної статті з [RO_HELPER_V0.md §8](./RO_HELPER_V0.md#8-повна-структура-контенту-категорії-та-статті-v0): заповнений контент UK, звірені **`ipsc_refs`**, робоче **`primary_url`**, за потреби блок **ФПСУ** з джерелом.
3. Окремий рядок у **реєстрі пакета** (таблиця нижче або `content/ro-helper/INDEX.md`): slug, статус `draft` → `reviewed` → `published`, дата рев’ю, рев’юер.

До початку масової підготовки **один раз** зафіксувати в реєстрі або в першій статті пакета поле **`ipsc_edition`** (рік/назва редакції IPSC handgun), **однакове для всього пакета v0**.

---

## 2. Вхідні дані (що мати перед стартом)

| Вхід | Вимога |
|------|--------|
| Офіційний PDF / сторінка **IPSC** handgun | Актуальна редакція, з якої знімаються номери пунктів |
| Документи **ФПСУ** (за потреби локального шару) | URL або PDF для цитування в `fpsu_refs` / блоці «Локально» |
| Таблиця статей | [RO_HELPER_V0.md §8](./RO_HELPER_V0.md#8-повна-структура-контенту-категорії-та-статті-v0) — slug не змінювати без узгодження (URL-стабільність) |
| Рев’юер | Людина, яка підписує статус `reviewed` (власник продукту або делегат) |

**Не потрібно:** завантажувати повні PDF у репозиторій; достатньо посилань у frontmatter і в тексті.

## 3. Структура каталогів (рекомендація до імплементації)

Поки немає коду модуля, зручно тримати чернетки в репо:

```text
content/ro-helper/
  INDEX.md              # реєстр: slug, title, category, status, reviewer, дата
  uk/
    safety/
      break-180.md
      ...
    penalties/
      ...
```

Після появи `src/features/ro-helper/` шлях можна перенести або залишити `content/ro-helper` як джерело правди — узгодити з розробкою в одному коміті.

## 4. Порядок виконання (по одній статті)

### Крок A — Підготовка контексту

1. Відкрити PDF IPSC на відповідному розділі (за **`ipsc_refs`** з [§8](./RO_HELPER_V0.md#8-повна-структура-контенту-категорії-та-статті-v0)).
2. Скопіювати **точну** нумерацію пунктів і переконатися, що вона відповідає обраній **`ipsc_edition`**.
3. Якщо потрібен блок ФПСУ — відкрити відповідний документ ФПСУ або зафіксувати «відмінностей немає / не знайдено» (тоді блок опускається).

### Крок B — Чернетка (LLM)

**Промпт-рамка (скопіювати й підставити поля):**

```text
Ти допомагаєш підготувати статтю для RO Helper (Shooters Tools) — довідник для Range Officers.
Мова статті: українська. Дисципліна v0: handgun (+ PCC лише якщо тема стосується).
Не вигадуй номери пунктів IPSC. Використовуй лише такі орієнтири з нашого списку: {ipsc_refs з RO_HELPER_V0 §8 для цього slug}.
Редакція правил (заповни з PDF): {ipsc_edition}.

Завдання:
1) Короткий заголовок і «Що це» (2–4 речення) для RO на лінії.
2) Розділ «Що робить RO» — буллети, практичні кроки.
3) Розділ «IPSC» — номери пунктів + посилання primary_url (я дам плейсхолдер, підстав офіційний URL з ipsc.org або PDF).
4) Опційно «Локально (ФПСУ)» — лише якщо є реальна відмінність; інакше напиши "Немає окремого блоку".
5) «Типові помилки / плутанина» — 1–3 пункти.
Не цитуй довгі абзаци rulebook дослівно; перефразуй коротко + вкажи пункти.

Slug статті: {slug}
Категорія frontmatter: {category}
```

Після відповіді моделі **зберегти** як `.md` і додати **frontmatter** вручну (моделі часто плутають YAML).

### Крок C — Рев’ю (людина, обов’язково)

Пройти [чекліст з RO_HELPER_V0 §2.1](./RO_HELPER_V0.md#21-чекліст-ревю-перед-релізом-статті):

- [ ] Кожен пункт IPSC звірено з PDF.
- [ ] `primary_url` відкривається і відповідає редакції.
- [ ] Немає «вигаданих» штрафів/очок без посилання на таблицю (особливо **scoring**).
- [ ] Для **Safety**: за можливості вибіркова перевірка другим досвідченим RO.
- [ ] `status` змінено на `reviewed`, заповнено `reviewer`, `review_date`.

### Крок D — Публікація

`status: published` лише коли стаття **підключена** в застосунку (або явно схвалена до релізу білду). До інтеграції в UI достатньо `reviewed`.

## 5. Структура тіла статті (обов’язкові розділи UK)

Рекомендований порядок блоків у Markdown (після frontmatter):

1. `## Що це` — суть для RO.
2. `## Що робить RO` — дії на лінії.
3. `## IPSC` — пункти + посилання (primary).
4. `## Локально (ФПСУ)` — опційно; якщо немає — розділ не вставляти.
5. `## Типові помилки` — опційно, але бажано.

Мінімальна довжина: **орієнтир 300–800 слів** на статтю (не роздувати до підручника).

---

## 6. Критерії приймання статті

Стаття вважається **прийнятою** (`reviewed`), якщо:

1. Frontmatter містить усі **обов’язкові** поля: `title`, `slug`, `category`, `locale: uk`, `ipsc_edition`, `ipsc_refs`, `primary_url`, `reviewer`, `review_date`, `status`.
2. `slug` **збігається** з таблицею [RO_HELPER_V0 §8](./RO_HELPER_V0.md#8-повна-структура-контенту-категорії-та-статті-v0) (або узгоджене відхилення задокументоване в `INDEX.md`).
3. Жодних числових значень скорингу без **підтвердження** таблицею з PDF (для категорії `scoring`).
4. Мова — **українська**, без змішування з англійською в заголовках розділів (англійські терміни IPSC в дужках дозволені).

Оновлювати рядок одразу після рев’ю.

## 9. Зв’язок з розробкою

Після набору статей у `reviewed` розробник:

- підключає шляхи до файлів або імпорт у бандл (див. [RO_HELPER_V0.md §9](./RO_HELPER_V0.md#9-технічна-рамка-імплементація-після-спеки));
- не змінює **`slug`** без редіректу / міграції.

Питання по UX (Quick Cite, шар ФПСУ) — лише з [RO_HELPER_V0.md](./RO_HELPER_V0.md).

---

## 10. Типові помилки (не робити)

- Публікувати текст моделі **без** перевірки PDF.
- Копіювати **цілі сторінки** rulebook у статтю.
- Змінювати slug «для краси» після того, як хтось уже послався на URL.
- Ставити конкретні очки Charlie/Delta **лише для Minor**, не згадуючи Major (для handgun).
- Писати блок ФПСУ «навмання» без джерела в `fpsu_refs`.
