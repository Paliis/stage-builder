# ТЗ: підготовка статей RO Helper (v0)

**Призначення:** операційне завдання для виробництва Markdown-статей модуля **RO Helper** за специфікацією [RO_HELPER_V0.md](./RO_HELPER_V0.md). Пайплайн: **модель C** — чернетка з LLM, **обов’язкове рев’ю** людиною перед публікацією.

**Останнє оновлення:** квітень 2026 (UK+EN, дисципліни, ФПСУ — хто верифікує).

---

## 1. Результат роботи (deliverables)

1. Для **кожного** `slug` з [RO_HELPER_V0.md §8](./RO_HELPER_V0.md#8-повна-структура-контенту-категорії-та-статті-v0) — **два** файли **`*.md`** (`locale: uk` та `locale: en`) з валідним **YAML frontmatter** ([§2.2](./RO_HELPER_V0.md#22-поля-контенту-рекомендований-frontmatter)): однаковий `slug`, `discipline`, `ipsc_refs`, узгоджені `primary_url` (зазвичай однакові для пари мов).
2. Звірені пункти IPSC з **правильного** PDF для **`discipline`** (v0: переважно handgun; rifle/shotgun — лише коли явно розширюєте пакет, див. [RO_HELPER_V0.md §6](./RO_HELPER_V0.md#6-дисципліни-види-стрільби-та-документи-ipsc)).
3. Блок **«Локально (ФПСУ)»** — лише якщо виконано [RO_HELPER_V0.md §4.1](./RO_HELPER_V0.md#41-хто-фіксує-відмінності-фпсу-від-ipsc); інакше розділ відсутній, `fpsu_delta_verified: false`.
4. Рядок у **`content/ro-helper/INDEX.md`**: slug, `discipline`, статуси **uk** / **en** окремо, `fpsu_delta_verified`, рев’юер, дати.

До старту пакета зафіксувати **`ipsc_edition`** (як мінімум для **handgun**; для інших дисциплін — окремо, коли з’являться статті).

---

## 2. Вхідні дані (що мати перед стартом)

| Вхід | Вимога |
|------|--------|
| Офіційні PDF / сторінки **IPSC** | Для кожної **дисципліни**, з якої берете пункти (v0: handgun; окремо PCC за структурою IPSC) |
| Документи **ФПСУ** | Для верифікації відмінностей; без них — не писати блок ФПСУ |
| Таблиця статей | [RO_HELPER_V0.md §8](./RO_HELPER_V0.md#8-повна-структура-контенту-категорії-та-статті-v0) — slug не змінювати без узгодження |
| Рев’юер | Підписує **обидві** мовні версії (можна одна дата, якщо рев’ю послідовне) |
| Хто порівнює ФПСУ з IPSC | **Той самий рев’юер** або узгоджений консультант; див. §4.1 спеки |

**Не потрібно:** завантажувати повні PDF у репозиторій; достатньо посилань у frontmatter і в тексті.

---

## 3. Структура каталогів (рекомендація до імплементації)

```text
content/ro-helper/
  INDEX.md
  uk/
    safety/
      break-180.md
    ...
  en/
    safety/
      break-180.md
    ...
```

Після появи `src/features/ro-helper/` шлях можна перенести або залишити `content/ro-helper` як джерело правди — узгодити з розробкою в одному коміті.

---

## 4. Порядок виконання (по одній статті)

### Крок A — Підготовка контексту

1. Визначити **`discipline`** (v0: зазвичай `handgun`) і відкрити відповідний **PDF IPSC**, не змішуючи з rifle/shotgun.
2. Знайти розділ за **`ipsc_refs`** з [§8](./RO_HELPER_V0.md#8-повна-структура-контенту-категорії-та-статті-v0); зафіксувати **`ipsc_edition`**.
3. **ФПСУ:** якщо плануєте блок «Локально» — відкрити **релевантний** документ ФПСУ і **вручну** відмітити, чи є відмінність; інакше блок не додавати.

### Крок B — Чернетка (LLM)

Згенерувати **дві** версії (спочатку UK, потім EN на основі узгодженого UK, або два окремі виклики). **Промпт-рамка:**

```text
Ти допомагаєш підготувати статтю для RO Helper (Shooters Tools).
Мова статті: {українська АБО англійська — одна на файл}.
Дисципліна: {discipline, v0 зазвичай handgun}. PDF IPSC для цієї дисципліни — єдине джерело номерів пунктів.
Не вигадуй номери. Орієнтири пунктів: {ipsc_refs з RO_HELPER_V0 §8 для slug}.
Редакція: {ipsc_edition}.

Завдання:
1) Заголовок і перший розділ (UK: «Що це» / EN: «What this is») — 2–4 речення.
2) Дії RO (UK: «Що робить RO» / EN: «RO actions»).
3) Розділ IPSC — пункти + primary_url (плейсхолдер замінити на офіційне посилання).
4) «Локально (ФПСУ)» / «Local (FPSU)» — ТІЛЬКИ якщо в контексті надано підтверджені витяги з документа ФПСУ; інакше явно: без цього розділу.
5) Типові помилки (1–3 пункти).
Не цитуй довгі абзаци rulebook дослівно.

Slug: {slug}
category: {category}
```

Після відповіді — **frontmatter** вручну; для EN-файлу `locale: en`, `title` англійською.

### Крок C — Рев’ю (людина, обов’язково)

Пройти [чекліст RO_HELPER_V0 §2.1](./RO_HELPER_V0.md#21-чекліст-ревю-перед-релізом-статті) **для UK і для EN** окремо:

- [ ] Пункти IPSC звірено з PDF **потрібної дисципліни**.
- [ ] UK і EN **узгоджені** за змістом (ті самі правила, та сама логіка ФПСУ).
- [ ] Блок ФПСУ: лише якщо рев’юер підтвердив за документом; тоді `fpsu_delta_verified: true`.
- [ ] `status: reviewed`, `reviewer`, `review_date` в **обох** файлах.

### Крок D — Публікація

`published` лише коли **обидві** мови у статусі `reviewed` (або вище) і модуль готовий їх показати.

---

## 5. Структура тіла статті

**UK-файл:** `## Що це`, `## Що робить RO`, `## IPSC`, опційно `## Локально (ФПСУ)`, `## Типові помилки`.

**EN-файл:** ті самі змістові блоки англійською, наприклад `## What this is`, `## RO actions`, `## IPSC`, опційно `## Local (FPSU)`, `## Common mistakes`.

Мінімальна довжина: **орієнтир 300–800 слів** на файл.

---

## 6. Критерії приймання пари статей

Пара **UK + EN** для одного `slug` вважається готовою до `published`, якщо:

1. У кожному файлі заповнені обов’язкові поля з [§2.2](./RO_HELPER_V0.md#22-поля-контенту-рекомендований-frontmatter), включно з `discipline`, коректним `locale`.
2. `slug` і `discipline` збігаються між файлами.
3. Для **scoring** — числа лише з первинника для цієї дисципліни.
4. Заголовки розділів **однією мовою** в межах файлу (UK або EN).

---

## 9. Зв’язок з розробкою

Після набору статей у `reviewed` розробник:

- підключає шляхи до файлів або імпорт у бандл (див. [RO_HELPER_V0.md §9](./RO_HELPER_V0.md#9-технічна-рамка-імплементація-після-спеки));
- не змінює **`slug`** без редіректу / міграції.

Питання по UX (Quick Cite, шар ФПСУ) — лише з [RO_HELPER_V0.md](./RO_HELPER_V0.md).

---

## 10. Типові помилки (не робити)

- Публікувати **лише UK** без EN (або навпаки) у фінальному релізі пари.
- Плутати **PDF handgun** з rifle/shotgun при номерах пунктів.
- Публікувати текст моделі **без** перевірки PDF.
- Копіювати **цілі сторінки** rulebook у статтю.
- Змінювати slug «для краси» після того, як хтось уже послався на URL.
- Ставити конкретні очки Charlie/Delta **лише для Minor**, не згадуючи Major (для handgun).
- Писати блок ФПСУ **без** перевірки рев’юером за офіційним документом ФПСУ (або покладатися на «модель каже, що відрізняється»).

---

## 11. Хто порівнює правила ФПСУ з IPSC

Коротко: **відповідальний рев’юер** (ви або делегат) **сам** звіряє офіційні тексти. LLM може прискорити **чернетку**, але не є джерелом істини. Деталі — [RO_HELPER_V0.md §4.1](./RO_HELPER_V0.md#41-хто-фіксує-відмінності-фпсу-від-ipsc).
