# BL-004 — Активації на плані (специфіка рішень)

**Статус:** узгоджені продуктові рішення для реалізації (кандидат до грумінгу в `candidate` / `ready`).  
**Зв’язок:** [BACKLOG.md](./BACKLOG.md) **BL-004**, [USER_FEEDBACK.md](./USER_FEEDBACK.md) (розділ про активації).

---

## 1. Нумерація

- **Глобальні номери** для всієї вправи: кожен об’єкт, який бере участь у зв’язках активації, має **один унікальний номер** у межах сцени (немає двох «активаторів з номером 2» тощо).
- Підписи **1 / 2 / …** на плані (і в 3D) відображають саме цю **глобальну** схему, а не локальні пари «завжди 1 і 2 на кожній лінії».

## 2. Типи об’єктів у зв’язку

- **Типово** активуються **мішені**.
- У **логіці даних** поки **не обмежуємо** кінці зв’язку лише мішенями: джерело й ціль можуть бути будь-якими підтримуваними сутностями сцени (мішені, реквізит тощо), поки це технічно узгоджено з моделлю `targets` / `props` та стабільними id.

## 3. Брифінг

- Текст про активації **потрапляє в той самий блок**, що й автоматичний опис складу мішеней — поле **`targetsDescription`** (те саме, що заповнюється кнопкою на кшталт «з сцени в брифінг» через `summarizeTargets` / аналог).
- **Логіка:** при оновленні зі сцени **доповнювати** (або збирати разом) опис складу мішеней **і** рядки про активації в **одному** проході, щоб користувач бачив узгоджений текст; після генерації текст **можна вільно редагувати** (як зараз для складу мішеней).
- Деталі формулювань — **i18n** (UK / EN), шаблони в `messages.ts`.

## 4. Мови

- Поведінка **однакова** для **обох** мов інтерфейсу; згенеровані фрази — через ті самі механізми локалізації, що й решта брифінгу.

## 5. Педаль / окремий тип «активатор»

- **Не входить** у перший скоп BL-004; **окрема задача** після базової моделі зв’язків (за потреби новий пункт беклогу або етап 2 BL-004).

## 6. 2D і 3D

- **2D:** лінія (стрілка) активації + **глобальні** числові підписи на об’єктах.
- **3D:** **те саме** — підписи та лінії активації, узгоджено з 2D (той самий набір зв’язків і номерів).

---

## Наступні технічні кроки (чернетка для реалізації)

- Модель даних: масив **ребер** `{ fromRef, toRef }` (посилання на екземпляри мішені/реквізиту) + правило присвоєння **глобальних номерів** учасникам графа.
- Файл вправи: розширення схеми та **`STAGE_PROJECT_VERSION`** / міграція.
- Режим редактора «Активація»: взаємодія в `StageCanvas`; відображення в `StageView3D`.
- Функція збірки тексту для **`targetsDescription`**: розширити або композиція з `summarizeTargets` + блок активацій.

*Деталі API полів і порядку речень у брифі — уточнювати при розбитті на задачі в спринті.*
