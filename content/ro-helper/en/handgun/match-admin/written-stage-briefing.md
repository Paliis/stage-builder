## What this is

**3.2** defines what a **written stage briefing (WSB)** must contain and how it binds the **COF**: order of engagement, positions, mandatory reloads, special targets, etc. The **RO applies the WSB on the bay**: if something is **not written**, general IPSC rules usually apply — not a private oral deal with the **MD** unless published per procedure.

## What the Range Officer (RO) does

1. **Read the WSB** before the squad arrives; if a line is unclear — **CRO/RM** **before** shooting.
2. **Do not add** personal requirements absent from the WSB and PDF.
3. **Answer competitor questions** with WSB text + general rules if within **COF** scope.
4. **Report** mismatches between WSB and built props — **RM** before the start.
5. **During the run**, interpret mandatory / procedural items **literally**.

## IPSC (Jan 2026)

- Anchor: **3.2**; align with **Chapter 1** (stage design) and **Chapter 10** (penalties for WSB breaches).

## Local (FPSU)

Translated WSB at FPSU events must match the approved text; changes only via **MD**. **`fpsu_delta_verified`**: **false** until **RO_HELPER_V0 §4.1** review.

## Common mistakes

- **Oral add-ons** that change the published **COF**.
- **Different WSB versions** in the RO folder vs the competitor pack.
- **Ignoring** a clear prop error “because three squads already shot it”.
